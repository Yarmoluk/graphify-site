Concepts (11)

ConceptDefinition (from thread)Graphify.md Position
Semantic Layer Schema-level abstraction. Defines what things mean and how they relate structurally. Ontologies, taxonomies, data models. Necessary foundation but not sufficient for AI reasoning.
Context Layer Dynamic, situational layer. What's relevant NOW for THIS query/decision. Temporal, user-aware, task-specific. This is what Graphify.md builds. The .md delivery = context, not schema.
Knowledge Graph Entity-relationship structure. Joseph Davis: "All correct KGs are by definition context graphs." Our substrate. The thing we build that enables the context layer.
Context Graph Kurt Cagle: "Graph-based log of reified events." Dynamic world model. Decision support. Aligns with ADA's Decision Bots layer.
Reification Making statements about statements. "This relationship has confidence X, was asserted by Y, at time Z." Maps to Graphify's explainability layer. Audit trails.
Tacit Knowledge Jessica Talisman: Knowledge in people's heads. Requires elicitation framework + HITL + auditability. This is EXACTLY what Graphify captures. The estimator's judgment. The sales rep's instinct. The thing that retires when they do.
Proprietary Fragmentation Lulit/Younis warning: if context layers become proprietary, we recreate the silo problem one layer up. Graphify uses open standards (.md, MCP, Neo4j). This is a positioning advantage.
Entity Authority Younis/SHAMIL: declaring at the entity level what something is authorized to do, under what governance. Governance gap we haven't addressed yet. Potential integration point.
Non-determinism Younis: without declared entity authority, agents must INFER governance — that's where non-determinism returns. SHAMIL solves notation. Graphify solves structure. Complementary.
Agentic Memory Pavlyshyn's book concept. Memory + cognitive processes + promise graphs for multi-agent systems. Adjacent to Graphify. Our graphs ARE the memory layer for agents.
GraphRAG Dan's comment: "I'm tired of being on the RAG, I want GraphRAG." 112 impressions. Our positioning. RAG retrieves. GraphRAG reasons.
